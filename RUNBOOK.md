# RUNBOOK для менеджера контенту

## 1) Простий сценарій для менеджера

1. Відкрити `/admin-guide`.
2. Натиснути `1. Відкрити адмінку (CMS)`.
3. Внести зміни у секції `Менеджер (базовий режим)`.
4. Натиснути `Save changes` у CMS.
5. Повернутись на `/admin-guide` і натиснути `2. Я зберегла зміни`.
6. У вікні обрати:
   - `Переглянути Preview`, якщо хочете спочатку перевірити.
   - `Опублікувати`, якщо чекліст відмічений і `Quality Gate` без помилки.

## 2) Що обов'язково перевірити перед публікацією

1. Головна сторінка (`/`)
2. Каталог ігор (`/igry`)
3. Список проєктів (`/kik/proekty`)
4. Одна сторінка гри (`/igry/[slug]`)
5. Одна сторінка проєкту (`/kik/proekty/[id]`)
6. Мобільний вигляд
7. Соціальні посилання

## 3) Якщо зламалось після публікації (швидкий rollback)

1. Відкрити `Actions` у GitHub.
2. Запустити workflow `Quick Rollback Main`.
3. Вставити SHA коміту, який треба відкотити.
4. Дочекатися автоматичного revert-коміту в `main`.
5. Створити Issue через шаблон `Content Incident`.

## 4) Правила безпеки

1. Не пушити напряму в `main`.
2. Не редагувати JSON вручну поза CMS, якщо не технічний користувач.
3. Публікацію робити тільки через кнопку `Опублікувати` на `/admin-guide`.
4. Якщо є сумніви щодо поля або URL, не публікувати без перевірки.
5. Для slug/мета-підказок використовувати `/admin-guide`.
