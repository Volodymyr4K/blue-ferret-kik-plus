# RUNBOOK для менеджера контенту

## 1) Як оновити контент без ризику

1. Відкрити Pages CMS.
2. Внести зміни у поля.
3. Зберегти зміни в окремий commit.
4. Створити PR у `main`.
5. Дочекатися зеленого `Quality Gate`.
6. Попросити рев'ю і тільки тоді merge.

## 2) Що обов'язково перевірити перед merge

1. Головна сторінка (`/`)
2. Каталог ігор (`/igry`)
3. Список проєктів (`/kik/proekty`)
4. Одна сторінка гри (`/igry/[slug]`)
5. Одна сторінка проєкту (`/kik/proekty/[id]`)
6. Мобільний вигляд
7. Соціальні посилання

## 3) Якщо зламалось після публікації

1. Відкрити останній робочий commit у GitHub.
2. Натиснути `Revert`.
3. Створити PR з revert.
4. Після зеленого `Quality Gate` зробити merge.
5. Занести причину інциденту в PR-коментар (1-2 речення).

## 4) Правила безпеки

1. Не пушити напряму в `main`.
2. Не редагувати JSON вручну поза CMS, якщо не технічний користувач.
3. Усі зміни лише через PR.
4. Якщо є сумніви щодо поля або URL, не публікувати без перевірки.
